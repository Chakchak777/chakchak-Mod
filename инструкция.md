1 Дар Котости — поговорить с Котостью и получить алмаз (quest4)
2 Охота на нежить — убить зомби (quest5)

Каждый квест состоит из двух JSON файлов, записи в QuestDefinitions и двух вызовов в Java (unlock и complete)

Общая схема 

questX_unlocked + unlock() → квест появляется в меню (R)
questX_completed + complete() → квесту ставится галочка ✓

| Действие | Метод | Когда вызывать |
| Открыть квест | QuestAdvancements.unlock(player, questX) | Игрок может выполнить цель |
| Закрыть квест | QuestAdvancements.complete(player, questX) | Цель выполнена |

Всё вызывается только на сервере (ServerPlayer)

Шаг 5 Закрыть квест при убийстве — complete

В том же ModServerEvents java

import net minecraft world entity EntityType

Обработчик смерти

SubscribeEvent
public static void onLivingDeath(LivingDeathEvent event) {
LivingEntity victim = event getEntity()
if (victim getType() != EntityType ZOMBIE) {
return
}

ServerPlayer killer = getKillingPlayer(event getSource())
if (killer != null) {
QuestAdvancements complete(killer, quest5)
}
}

private static ServerPlayer getKillingPlayer(DamageSource source) {
Entity directEntity = source getEntity()
if (directEntity instanceof ServerPlayer player) {
return player
}
Entity causingEntity = source getDirectEntity()
if (causingEntity instanceof ServerPlayer player) {
return player
}
return null
}

Учитывается убийство мечом, луком, стрелой с лука и т д — если урон от игрока

Если что-то не работает

| Симптом | Что проверяю |
| Нет кнопки в меню R | Не вызван unlock, нет JSON, опечатка в id |
| Кнопка есть, нет ✓ | Не вызван complete |
| Команда grant не работает | Путь chakchakmod quests quest5 unlocked |
| quest4 не открывается | Сценарий Котоста = 3 (индекс 2) |
| quest5 не закрывается | Убил ли игрок (не голем, падение) |

Шпаргалка на одну строку

QuestAdvancements complete(serverPlayer, quest id); поставить ✓
